# BJTU Vision — 运行指南

## 项目简介

`bjtu-vision` 是计算机视觉课程的两项任务实现：

| 任务 | 脚本 | 说明 |
|------|------|------|
| 图像检索 | `retrieval_v2.py` | ResNet50 特征提取 + 余弦相似度检索，输出 P@K 曲线 |
| 文字检测 | `detection_v2.py` | LabelMe 转 YOLO，YOLOv8n 训练+推理，可视化检测框 |
| 检索演示 | `demo_retrieval_show.py` | 随机选 query，展示检索结果并自动打开 |
| 检测演示 | `demo_detection_show.py` | 随机选 query，展示检索结果+文字检测框并自动打开 |

---

## 1. 环境准备

### 依赖安装

```bash
pip install -r requirements.txt
```

依赖列表：torch>=2.0, torchvision>=0.15, ultralytics>=8.0, numpy, scikit-learn, matplotlib, pillow, PyYAML。

### 硬件要求
- 推荐 GPU（脚本自动检测 CUDA，否则回退 CPU）
- 纯 CPU 运行很慢，尤其 YOLO 训练阶段
- 测试环境：Python 3.10+, CUDA 12.1

---

## 2. 数据集结构

数据集根路径硬编码在各脚本的 `DATASET_ROOT` 中（默认 `E:\playground\交大视觉印象数据集2026`），必须包含：

```
交大视觉印象数据集2026/
├── image_retrieval/
│   ├── query/                    # 查询图片
│   └── base/
│       ├── BJTU/                 # 北京交大 base 图片
│       └── util_pic/             # 公用 base 图片
└── object_detection/
    └── data/                     # 图片 + LabelMe JSON 标注
```

> 如数据集放在其他路径，修改对应脚本中的 `DATASET_ROOT` 即可。

---

## 3. 运行顺序

脚本之间有依赖关系，**必须按以下顺序执行**：

```
retrieval_v2.py ───────────────┐
    │                          │
    └── 产出: retrieval_results.json ─┐
                                       │
detection_v2.py ───────────────────────┤（同时需要上一步的 JSON）
    │                                  │
    └── 产出: best.pt（YOLO 模型）      │
                                       │
---------------------------------------
    │
    ├── demo_retrieval_show.py（只需 retrieval_results.json）
    └── demo_detection_show.py（需要 JSON + best.pt）
```

---

### 3.1 步骤一：图像检索

```bash
python retrieval_v2.py
```

**流程：**

1. 收集 base（BJTU + util_pic）和 query 图片
2. 用预训练 ResNet50 提取 2048 维特征，L2 归一化
3. 计算 query × base 余弦相似度矩阵并排序
4. 按 12 个 landmark（fhy/jx/kx/mh/nm/sjz/sy/tsg/ty/yf/yk/zx）分别评估 P@20 / P@40 / P@60
5. 输出 P@K 曲线图和检索结果 JSON

**输出位置：** `outputs/retrieval_v2/`

| 文件 | 说明 |
|------|------|
| `base_features.npy` | base 图片特征缓存 |
| `query_features.npy` | query 图片特征缓存 |
| `retrieval_results.json` | 每个 query 的 top-10 检索图片路径 |
| `P@K_{landmark}.png` | 每个 landmark 的 P@K 曲线（12 张） |

**缓存：** `.npy` 文件存在时跳过特征提取。如需强制重新提取，删除对应 `.npy` 文件即可。

---

### 3.2 步骤二：文字检测

```bash
python detection_v2.py
```

**流程：**

1. **格式转换** — 把 `object_detection/data/` 下的 LabelMe JSON 标注转换为 YOLO 格式（`outputs/detection_v2/yolo_dataset/`）
   - 按 8:2 划分训练集/验证集
   - 数据清洗：过滤面积 < 0.05%、边长 < 0.5%、宽高比 > 20:1 的异常框
2. **训练 YOLOv8n** — 50 epoch，含 label smoothing、mosaic、HSV 增强等
3. **检测 + 可视化** — 读取 `retrieval_v2` 的输出，对 query + top-10 图片做文字检测，画红框

**输出位置：** `outputs/detection_v2/`

| 文件/目录 | 说明 |
|------|------|
| `yolo_dataset/` | 转换后的 YOLO 训练数据 |
| `train/weights/best.pt` | 训练好的 YOLOv8n 模型权重 |
| `detections_all.json` | 所有图片的文字检测结果缓存 |
| `detection_{landmark}_{index}.png` | 检测可视化图（2×5 网格，红框标出文字） |

**缓存：**
- `data.yaml` 存在 → 跳过格式转换
- `best.pt` 存在 → 跳过训练
- `detections_all.json` 存在 → 跳过检测推理

---

### 3.3 步骤三（可选）：检索结果演示

```bash
python demo_retrieval_show.py
```

**前提：** 已运行 `retrieval_v2.py`，产出 `outputs/retrieval_v2/retrieval_results.json`。

**效果：**
1. 随机选取 4 个不同 landmark 的 query 图片
2. 对每个 query 生成一张展示图：左侧为 query 原图，右侧为 2×5 的 top-10 检索结果网格
3. 保存后用系统默认图片查看器自动打开（Windows `os.startfile`）

**输出位置：** `outputs/demo_retrieval_show/`

| 文件 | 说明 |
|------|------|
| `demo_retrieval_{landmark}.png` | 检索结果展示图（4 张） |

---

### 3.4 步骤四（可选）：检测结果演示

```bash
python demo_detection_show.py
```

**前提：** 已运行 `retrieval_v2.py` 和 `detection_v2.py`（需要 `retrieval_results.json` + `best.pt`）。

**效果：**
1. 随机选取 4 个不同 landmark 的 query 图片
2. 对每个 query 的 top-10 检索结果图片跑 YOLO 文字检测
3. 生成 2×5 网格图，检测到的文字区域用红框标注
4. 保存后自动打开

**输出位置：** `outputs/demo_detection_show/`

| 文件 | 说明 |
|------|------|
| `demo_detection_{landmark}.png` | 检测结果展示图（4 张） |

---

## 4. 一键运行

```bash
python retrieval_v2.py && python detection_v2.py && python demo_retrieval_show.py && python demo_detection_show.py
```

首次运行预计耗时（GPU）：
- 检索部分：5-10 分钟
- 检测训练部分：20-40 分钟

后续运行利用缓存，仅需数分钟。

## 5. 常见问题

**Q: 报错 `No module named 'ultralytics'`**  
A: 执行 `pip install ultralytics` 或 `pip install -r requirements.txt`。

**Q: 数据集路径不对**  
A: 编辑对应脚本中的 `DATASET_ROOT` 变量为你的实际路径。

**Q: demo 脚本没有自动打开图片**  
A: 底层使用 `os.startfile`，仅 Windows 生效。其他系统会保存图片但不会自动打开。

**Q: 如何强制重新运行（不使用缓存）**  
A: 删除对应的缓存文件：
- 重新提取特征 → 删除 `outputs/retrieval_v2/*.npy`
- 重新训练模型 → 删除 `outputs/detection_v2/train/`
- 重新检测 → 删除 `outputs/detection_v2/detections_all.json`

**Q: GPU 内存不足**  
A: 减小 `detection_v2.py` 中的 `YOLO_BATCH`（默认 16，可改为 8 或 4）。

**Q: `retrieval_v2.py` 和 `retrieval.py` 有什么区别？**  
A: `retrieval_v2.py` 是改进版，增加了特征缓存、P@K 曲线按 landmark 分组输出、检索结果 JSON 导出等功能。`retrieval.py` 为基础版本。

**Q: `detection_v2.py` 和 `detection.py` 有什么区别？**  
A: `detection_v2.py` 是改进版，增加了更完善的数据清洗（过滤异常框）、检测结果缓存、以及更细粒度的训练增强参数。
